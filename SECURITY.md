# Security Policy

## Reporting a Vulnerability

We take the security of our software seriously. If you believe you have found a security vulnerability in our project, please report it to us via private email or GitHub Security Advisory.

### Private Email
Send an email to [security@agenticaiproductionsystem.com](mailto:security@agenticaiproductionsystem.com) with a detailed description of the vulnerability. Please do not disclose the vulnerability publicly until we have had a chance to address it.

### GitHub Security Advisory
Alternatively, you can create a private GitHub Security Advisory by following these steps:
1. Go to the [Security tab](https://github.com/agentic-ai-production-system/security) of our repository.
2. Click on "New private vulnerability report".
3. Provide a detailed description of the vulnerability.

We will acknowledge your report within 48 hours and provide an estimated timeline for addressing the issue.

## Supported Versions

We currently support the following versions of our project:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| 0.9.x   | :x:                |

If you are using a version that is not listed as supported, we strongly recommend upgrading to the latest supported version to ensure you receive security updates and patches.

## Response Timeline

We aim to respond to all security vulnerability reports within 48 hours. Once we have acknowledged the report, we will provide an estimated timeline for addressing the issue. Our goal is to provide a fix within 7 days of acknowledging the report.

## Hall of Fame

We are committed to recognizing the efforts of security researchers who help us improve the security of our project. We will publicly acknowledge and thank researchers who report valid, non-duplicate vulnerabilities that are fixed in our code.

## Out of Scope

The following items are considered out of scope for our security policy:

- Vulnerabilities in third-party libraries or dependencies. We recommend checking the security policies of these libraries for reporting vulnerabilities.
- Denial of Service (DoS) attacks that do not result in a compromise of the system.
- Social engineering attacks or phishing attempts.
- Vulnerabilities in the underlying infrastructure (e.g., cloud providers, hardware) that are not directly related to our project.

Thank you for helping us keep our project secure!
