# Security Policy

## Reporting a Vulnerability

We take the security of our software seriously. If you believe you have found a security vulnerability in our project, please report it to us as described below.

### Private Email
You can send an email to our security team at [security@agenticai.com](mailto:security@agenticai.com). Please include the following information:
- A description of the vulnerability
- Steps to reproduce the vulnerability
- Any potential impact of the vulnerability
- Your name and contact information (optional)

### GitHub Security Advisory
Alternatively, you can create a private GitHub Security Advisory for this repository. Please follow these steps:
1. Go to the [Security tab](https://github.com/agentic-ai-production-system/security) of this repository.
2. Click on "Report a vulnerability".
3. Fill out the form with as much detail as possible.

We will acknowledge your report within 48 hours and provide a timeline for the fix.

## Supported Versions

We currently support the following versions of our project:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| 0.9.x   | :x:                |
| < 0.9   | :x:                |

## Response Timeline

We aim to respond to all security reports within 48 hours. Once we have acknowledged the report, we will provide a timeline for the fix, which we aim to complete within 7 days.

## Hall of Fame

We are committed to recognizing the efforts of security researchers who help us improve our software. We will publicly acknowledge the names of researchers who have reported valid vulnerabilities that have been fixed.

## Out of Scope

The following items are out of scope for our security policy:

- Reports from automated tools without clear impact
- Reports of vulnerabilities in third-party libraries (unless they are directly related to our project)
- Reports of vulnerabilities that have already been publicly disclosed
- Reports of vulnerabilities that are not relevant to our project
